1. Introducción a n8n

¿Qué es n8n?

n8n es una plataforma de automatización de workflows que permite conectar aplicaciones y automatizar procesos sin escribir código.

Definición: "n8n" = "nodemation" (node + automation)

Características Principales

  • 🧩 Visual: Interfaz gráfica de arrastrar y soltar
  • 💻 Flexible: Open source y extensible
  • ⚡ Poderoso: 400+ integraciones nativas

Casos Típicos de Uso

  • 📧 Email → Sheets: Automatizar respuestas y guardar logs
  • 📱 Form → WhatsApp: Notificaciones instantáneas
  • 🤖 AI → Respuesta: Procesamiento inteligente
  • ⏰ Timer → Acción: Tareas programadas

2. Conceptos Fundamentales de n8n

Workflow (Flujo de Trabajo)

Un workflow es una secuencia de pasos automatizados que procesan datos y ejecutan acciones.

Componentes de un Workflow:

  • Trigger: Inicia el workflow
  • Nodos: Procesan o transforman datos
  • Conexiones: Flujo de datos entre nodos
  • Ejecuciones: Instancias del workflow

Ejemplo de Workflow: 

Email nuevo → Extraer datos → Guardar en Sheets → Notificar por Slack

Nodos en n8n

Los nodos son los bloques de construcción de cualquier workflow en n8n.

Tipos principales:

  • Trigger Nodes: Inician workflows
  • Regular Nodes: Procesan datos
  • Core Nodes: Funciones básicas

Ejemplos por categoría:

  • 📧 Comunicación: Gmail, Slack, Discord
  • 📊 Datos: Google Sheets, Airtable
  • 🤖 IA: OpenAI, Claude
  • ⚙️ Core: Set, IF, Function

Anatomía de un Workflow 

[TRIGGER] → [PROCESAR] → [GUARDAR] → [NOTIFICAR]
  1. TRIGGER: Detecta evento (email, webhook, timer)
  2. PROCESAR: Transforma o analiza datos
  3. GUARDAR: Almacena resultados
  4. NOTIFICAR: Envía confirmación o alerta

3. Tipos de Nodos en n8n

Trigger Nodes

Inician la ejecución del workflow

Nodos Populares:

Nodo Descripción Dificultad
Schedule Trigger Ejecuta en horarios específicos Fácil
Webhook Recibe datos HTTP externos Medio
Gmail Trigger Detecta emails nuevos Fácil
Google Forms Trigger Formulario enviado Fácil

Regular Nodes

Procesan y transforman datos

Nodos de Comunicación:

Nodo Descripción Dificultad
Gmail Envía emails Fácil
Google Sheets Lee/escribe hojas Medio
WhatsApp Business API Difícil
OpenAI Procesa con IA Fácil

Core Nodes (Fundamentales)

Bloques Básicos:

Nodo Función Uso
Set Modifica datos Cambiar formato, agregar campos
IF Lógica condicional Filtrar por condiciones
Function JavaScript custom Cálculos complejos
Filter Filtra elementos Remover datos no deseados
Merge Combina datos Unir información de múltiples fuentes
HTTP Request Llamadas API Integrar servicios externos

Regla de Oro

1 Trigger + N Nodos = 1 Workflow completo

Siempre empieza con un trigger, luego agrega los nodos necesarios

5. Flujo de Datos en n8n

¿Cómo fluyen los datos?

Los datos pasan de nodo en nodo como fichas de dominó, transformándose en cada paso.

Estructura de Datos

Formato JSON 

{
  "json": {
    "nombre": "Juan",
    "email": "juan@email.com",
    "edad": 25
  }
}

Campos principales:

  • json: Datos principales del elemento
  • binary: Archivos (imágenes, PDFs, etc.)

Acceso a Datos

8. Configuración de Credenciales

🔐 Configuración Paso a Paso

La configuración de credenciales es lo más complicado para principiantes, pero esencial para cualquier workflow.

Google Sheets (MEDIO)

Pasos detallados:

  1. Google Cloud Console:

    • Crear proyecto nuevo
    • Habilitar Google Sheets API
    • Crear Service Account

  2. Descargar JSON:

    • Crear clave para Service Account
    • Descargar archivo JSON

  3. Compartir hoja:

    • Copiar email del Service Account
    • Compartir Google Sheet con ese email
    • Dar permisos de Editor

  4. En n8n:

    • Pegar contenido del JSON
    • Probar conexión

💡 Tip: El Service Account es como un "usuario robot" que puede acceder a tus hojas.

Gmail (FÁCIL)

Opción 1: App Password (Recomendado)

  1. Activar 2FA:

    • Ir a Google Account Security
    • Activar verificación en 2 pasos

  2. Crear App Password:

    • Buscar "App passwords"
    • Crear password para "n8n"
    • Copiar password generado

  3. En n8n:

    • Email: tu@gmail.com
    • Password: app password (no tu password normal)

Opción 2: OAuth2

Más seguro pero más complejo. Requiere configurar OAuth en Google Cloud Console.

⚠️ Importante: Nunca uses tu contraseña normal de Gmail.

WhatsApp Business (DIFÍCIL)

Pasos para WhatsApp Business API:

  1. Meta Developer:

    • Crear cuenta en developers.facebook.com
    • Crear nueva App
    • Agregar producto WhatsApp

  2. Configurar número:

    • Verificar número de teléfono
    • Obtener Phone Number ID
    • Generar Access Token

  3. En n8n:

    • Phone Number ID
    • Access Token
    • Webhook URL (para recibir)

💰 Costo: WhatsApp Business API tiene costos por mensaje enviado.

OpenAI (FÁCIL)

Configuración simple:

  1. Ir a platform.openai.com
  2. API Keys → Create new
  3. Copiar clave (sk-...)
  4. Pegar en n8n

💳 Requiere: Tarjeta de crédito para uso

Buenas Prácticas de Seguridad

  • ✅ Nunca compartas tus API keys
  • ✅ Usa nombres descriptivos para credenciales
  • ✅ Prueba conexión antes de usar
  • ✅ Rota claves regularmente

Orden Recomendado para Aprender

  1. Gmail (App Password)
  2. Google Sheets (Service Account)
  3. OpenAI (API Key)
  4. WhatsApp (Business API)

9. Proyecto Práctico 1: Email Automático

📧 Tu Primer Workflow Real

Objetivo: Responder emails automáticamente y guardar log en Google Sheets

¿Qué vamos a hacer?

Cuando llegue un email con "pedido" en el asunto:

  1. Responder automáticamente
  2. Guardar los datos en Google Sheets

¿Por qué es útil?

  • ✅ Respuesta inmediata 24/7
  • ✅ Base de datos automática
  • ✅ Reduce trabajo manual
  • ✅ Mejora experiencia cliente

Nodos Necesarios

  1. Gmail Trigger 🟢 - Detecta emails nuevos
  2. IF Node 🔵 - ¿Contiene "pedido"?
  3. Gmail Send 🟣 - Envía respuesta
  4. Google Sheets 🟠 - Guarda registro

Paso a Paso Detallado

PASO 1: Gmail Trigger 

- Agregar Gmail Trigger
- Configurar credenciales
- Poll: Every 1 minute
- Filter: is:unread
- Probar ejecución

PASO 2: IF Node 

- Condition: String
- Value 1: {{ $json["subject"] }}
- Operation: contains
- Value 2: "pedido"
- Probar con email real

PASO 3: Gmail Send 

- To: {{ $json["from"]["value"][0]["address"] }}
- Subject: "RE: {{ $json["subject"] }}"
- Message: Plantilla de respuesta
- Conectar desde IF → True

PASO 4: Google Sheets 

- Operation: Append
- Document ID: URL de tu sheet
- Columns: Fecha, De, Asunto
- Conectar en paralelo con Gmail

Plantilla de Email 

Hola,

Hemos recibido tu pedido correctamente.

Número de orden: #{{ Math.floor(Math.random() * 10000) }}
Asunto: {{ $node["Gmail Trigger"].json["subject"] }}

Te contactaremos pronto con los detalles.

Gracias por tu compra!

Saludos,
Equipo de Ventas

Estructura Google Sheets

Fecha De Asunto
{{ $now }} {{ $json["from"] }} {{ $json["subject"] }}

💡 Tip: Crea las columnas antes en Google Sheets

10. Proyecto Práctico 2: WhatsApp Bot con IA

🤖 Bot Inteligente de WhatsApp

Objetivo: Crear un bot que responde mensajes automáticamente usando ChatGPT

Funcionalidades del Bot

  • 🧠 IA Conversacional: Responde preguntas naturalmente usando ChatGPT
  • 📊 Base de Conocimiento: Guarda todas las conversaciones en Google Sheets
  • ⚡ Tiempo Real: Respuesta inmediata 24/7
  • 🎯 Contextual: Recuerda información del cliente

Arquitectura del Bot 

WhatsApp Trigger → Procesar Mensaje → OpenAI (ChatGPT) → Guardar en Sheets
                                                        ↓
                                           Responder WhatsApp

Configuración Técnica

WhatsApp Trigger Setup 

Updates to listen for: message
Additional Fields:
- Include webhook validation
- Message filters: text
- Exclude: status messages

OpenAI Configuration 

Resource: Chat
Model: gpt-3.5-turbo
Max Tokens: 150
Temperature: 0.7
System Message: "Eres un asistente útil
de atención al cliente. Responde en 
español de forma amigable y concisa."

Flujo de Datos

1. Entrada (WhatsApp) 

{
  "from": "+549123456789",
  "message": "¿Cuál es el horario de atención?",
  "timestamp": "14:30:45"
}

2. Procesamiento (OpenAI) 

{
  "prompt": "Responde sobre horarios de atención",
  "model": "gpt-3.5-turbo",
  "response": "Nuestro horario de atención es de lunes a viernes de 9:00 a 18:00 hs. ¿En qué más puedo ayudarte?",
  "tokens": 45
}

3. Log (Google Sheets)

Fecha Usuario Pregunta Respuesta
14:30 +549... Horario? Lun-Vie...

4. Respuesta (WhatsApp) 

Para: +549123456789
Mensaje: "Nuestro horario de atención es de lunes a viernes de 9:00 a 18:00 hs. ¿En qué más puedo ayudarte?"
Estado: ✅ Enviado

  • 📊 Monitoreo: Logs detallados

Métricas del Workflow

  • ⏱️ Tiempo Total: ~2 segundos
  • 💰 Costo OpenAI: $0.001 por mensaje
  • 📊 Eficiencia: 100% automatizado

11. Buenas Prácticas y Debugging

🛠️ Tips de Experto

Buenas Prácticas

▶️ Ejecución Paso a Paso

Siempre ejecuta cada nodo individualmente antes de conectar todo el workflow.

🏷️ Nombres Descriptivos

Usa nombres claros para nodos:

  • "Gmail - Detectar Pedidos"
  • "Sheets - Guardar Cliente"
  • "WhatsApp - Notificar Venta"

🛡️ Manejo de Errores

  • Activa "Continue on Fail" en nodos críticos
  • Agrega nodos específicos para error handling
  • Implementa retry logic para APIs externas

👁️ Logs Detallados

  • Guarda logs de todas las ejecuciones
  • Incluye timestamps y IDs únicos
  • Registra tanto éxitos como errores

Errores Más Comunes

❌ Credenciales Incorrectas

Problema: API keys, tokens y permisos mal configurados
Solución: Verificar credenciales y probar conexión

❌ Sintaxis de Expresiones

Problema: Mal uso de {{ $json["campo"] }}
Solución: Revisar sintaxis y estructura de datos

❌ Rate Limits

Problema: Demasiadas requests muy rápido
Solución: Agregar delays entre requests masivos

❌ Data Mapping

Problema: Datos no llegan entre nodos
Solución: Revisar estructura de datos en cada paso

Herramientas de Debugging

🐛 Execution Log

Revisa cada ejecución, datos de entrada y salida de cada nodo

▶️ Execute Node

Ejecuta un solo nodo para probar su funcionamiento individualmente

📋 JSON View

Inspecciona la estructura exacta de los datos en formato JSON

Tips de Performance

  • ✅ Batching: Procesa múltiples items juntos
  • ✅ Caching: Evita llamadas API repetitivas
  • ✅ Filtros: Filtra datos temprano en el pipeline
  • ✅ Timeouts: Configura timeouts apropiados

Seguridad

  • 🔒 Never log API keys o passwords
  • 🔒 Webhook validation: Valida requests entrantes
  • 🔒 Rotate keys: Cambia credenciales regularmente
  • 🔒 Environment variables: Para datos sensibles

Checklist Pre-Producción

Configuración:

  • Todas las credenciales configuradas
  • Error handling implementado
  • Logs configurados apropiadamente
  • Pruebas con datos reales

Performance:

  • Rate limits considerados
  • Webhooks validados
  • Monitoreo configurado
  • Backup/rollback plan
